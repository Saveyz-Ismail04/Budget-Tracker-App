# Sultan's Treasury v5.0 🏛️
### Autonomous Personal Finance Ledger & Sandbox Validation System

Sultan's Treasury is a highly optimized, single-file client-side FinTech application designed to manage multi-month personal liquidity pipelines, track isolated savings targets via a secure "Imperial Vault," and provide real-time strategic financial advice based on user spending velocity. 

This application was engineered using an **AI-Assisted Systems Architecture workflow**, where I acted as the Product Manager and System Architect, directing an advanced Large Language Model (LLM) through 5 major structural iterations to build, test, and ship a functional, production-ready product.

## 🚀 Key Architectural Features
* **Time-Series Horizon Isolation:** Solved a critical data-overlap vulnerability from v4.0 by implementing categorical monthly timelines, ensuring that rolling balances do not bleed across active period boundaries.
* **Deterministic Cash-Flow Ingestion:** Features highly responsive input validation tracking using integer truncation (`Math.trunc()`), neutralizing floating-point index calculation errors common in open ledger arrays.
* **The Imperial Vault Protocol:** A specialized capital redirection mechanism that completely isolates designated liquidity rows out of daily spending risk paths, instantly recalibrating liquid available cash values.
* **Autonomous Advisor Matrix:** A built-in behavioral analytics engine that parses real-time cash flow margins and triggers dynamic system status warnings (e.g., automated over-allocation faults or duplicate lifestyle logging flags).
* **Premium Client-Side Persistence:** Utilizing unified local storage mapping (`localStorage`) to guarantee instant, secure state preservation directly on the user's local disk without server-side latency.

## 🛠️ Technology Stack & Engineering Methods
* **Core Computational Logic:** Pure JavaScript (ES6+) managing localized state JSON variables.
* **Visual Interface Assembly:** Responsive HTML5/CSS3 utilizing fluid CSS Grid and custom UI glow matrices optimized for high-refresh-rate displays.
* **Development Methodology:** Iterative Agile Sprint Framework (6 total version updates from conceptual draft to stable distribution), driving deep Quality Assurance (QA) cycles, debugging unique identifier (`uid`) collision risks, and optimizing keydown event listeners (`Enter` key overrides) for seamless data entry.
